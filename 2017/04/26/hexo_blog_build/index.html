搭建hexo + github pages + 独立域名博客

2017-04-26

Hexo，一个快速、简洁且高效的博客框架，Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

一、环境准备

  • 安装nodejs
  • 安装git
  • 申请github账号
    • 新建仓库 name.github.io（name与账号同名）
    • 配置ssh

二、安装hexo

1. 切换npm源

npm config set registry https://registry.npm.taobao.org

2. 安装hexo

npm install -g hexo

3. 设置windows的path路径

右击电脑_>属性->高级系统设置->环境变量（N）…->系统变量（S）->path D:\java\runtime\nodejs\node_data\node_global

hexo
hexo.cmd
...

执行这条命令不报commond not found即可

hexo -v

4. 初始化hexo

hexo init

5. 安装hexo依赖包

npm install

6. 生成、启动服务

hexo generate
hexo server

# Debug方式启动本地服务
hexo s –debug

7. 查看hexo是否安装成功

http://localhost:4000/

三、完整搭建过程【github + 独立域名】

1. 本地初始化hexo环境

mkdir hexo_blog
cd hexo_blog
hexo init
npm config set registry https://registry.npm.taobao.org
npm install

2. github创建新new repository

新建仓库
登陆github，单击+号创建新仓库，比较重要的点是：

  • 必须以github.io结尾；
  • 前缀必须与Owner一致【owner.github.io】

本地生成rsa

ssh-keygen -t rsa -C "mail@abc.com"

windows下c:/Users/Administrator/.ssh/文件中会生成三个文件（Administrator是你的账户名）

  • known_hosts
  • id_rsa # 私钥
  • id_rsa.pub # 公钥 需要保存至github

github保存ssh key
单击头像->settings->SSH keys and GPG keys->SSH keys_单击[new SSH keys] 添加id_rsa.pub公钥的文本内容

3. 配置_config.yml

比较关键的是# Deployment那处需要跟自己的github新创建的repository对应起来

# Site
title: 博客
subtitle: 生活美好，美好生活
description: 在哪里跌倒 就在哪里躺下
author: Lee
language: zh-CN
timezone: Asia/Shanghai

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:fbli/fbli.github.io.git
  branch: master

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
## yilia  https://github.com/litten/hexo-theme-yilia/wiki/Yilia源码目录结构及构建须知
## landscape
theme: yilia

3. 域名绑定

1、 新建CNAME文件，放到hexo_blog\source\下，内容为不带www的域名如：ezsay.cn
2、 配置域名解析

  • CNAME www name.github.io
  • @ name.github.io
    问题：能转向github，但是显示404。初次发布后，停10分钟再试，githubPages需要一定的时间

4. 发布到git上

hexo d

hexo d -g

hexo g -d

5. 访问

  • http://fbli.github.io/
  • https://fbli.github.io/

6. 更换主题

# 更换主题前必须clean
hexo clean
cd hexo_blog\themes
git clone git@github.com:litten/hexo-theme-yilia.git yilia # 此处有坑，默认的主题文件夹名需要改一下
cd yilia
npm install
cd ../../
hexo g
hexo s
http://localhost:4000/
# 发布
hexo d

四、Hexo常用命令汇总

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本

Hexo命令简写

hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
